Front-End

O desenvolvimento da web de front-end é a prática de converter dados em uma interface gráfica, por meio do uso de HTML, CSS e JavaScript, para que os usuários possam visualizar e interagir com esses dados.

Ferramentas utilizadas para desenvolvimento de front-end

Ferramentas utilizadas para desenvolvimento de front-end Existem várias ferramentas e plataformas (wordpress, magento etc.) disponíveis que podem ser usadas para desenvolver o front-end de um site, e entender quais ferramentas são mais adequadas para tarefas específicas marca a diferença entre o desenvolvimento de um site invadido e um bem projetado , site escalável.

Linguagem de marcação de hipertexto (HTML)

Hyper Text Markup Language (HTML) é a espinha dorsal de qualquer processo de desenvolvimento de website, sem o qual não existe uma página web. Hipertexto significa que o texto possui links, denominados hiperlinks, incorporados nele. Quando um usuário clica em uma palavra ou frase que tenha um hiperlink, ele trará outra página da web. Uma linguagem de marcação indica que o texto pode ser transformado em imagens, tabelas, links e outras representações. É o código HTML que fornece uma estrutura geral de como o site ficará. HTML foi desenvolvido por Tim Berners-Lee. A última versão do HTML é chamada HTML5 e foi publicada em 28 de outubro de 2014 pela recomendação W3. Esta versão contém maneiras novas e eficientes de lidar com elementos como arquivos de vídeo e áudio.

Folhas de Estilo em Cascata (CSS)

Cascading Style Sheets (CSS) controla o aspecto de apresentação do site e permite que seu site tenha uma aparência única. Isso é feito mantendo-se as folhas de estilo que ficam no topo de outras regras de estilo e são acionadas com base em outras entradas, como o tamanho e a resolução da tela do dispositivo.

JavaScript

JavaScript é uma linguagem de programação imperativa baseada em eventos (em oposição ao modelo de linguagem declarativa do HTML) que é usada para transformar uma página HTML estática em uma interface dinâmica. O código JavaScript pode usar o [Document Object Model] (DOM), fornecido pelo padrão HTML, para manipular uma página da Web em resposta a eventos, como a entrada do usuário. Usando uma técnica chamada AJAX, o código JavaScript também pode recuperar ativamente o conteúdo da Web (independente da recuperação da página HTML original) e também reagir a eventos do lado do servidor, adicionando uma natureza verdadeiramente dinâmica à experiência da página da Web.

WebAssembly

O WebAssembly, suportado por todos os principais navegadores (ou seja, dos principais fornecedores Google, Apple, Mozilla e Microsoft), é a única alternativa ao JavaScript para executar código em navegadores da Web (sem a ajuda de plug-ins, como Flash, Java ou Silverlight; todos foram descontinuados, pois os navegadores estão removendo o suporte a plug-ins). Antes de sua adoção, havia asm.js (um subconjunto de JavaScript e, portanto, funciona estritamente em todos os navegadores), que também é usado como um destino de compilador com suporte eficiente em navegadores como o Internet Explorer 11; e para esses navegadores que não suportam o WebAssembly diretamente, ele pode ser compilado para o asm.js e para os navegadores suportados dessa maneira. De modo geral, os programadores não programam diretamente no WebAssembly (ou asm.js), mas usam linguagens como Rust, C ou C ++ ou, em teoria, qualquer idioma que compila para ele.
